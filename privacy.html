VideoLM Privacy Policy

Last updated: April 17, 2026

VideoLM does not collect, transmit, or sell any personal information. All data processing occurs locally in your browser.

1. Data We Collect

VideoLM processes the following data locally in your browser. None of this data is sent to our servers because we do not operate any servers.

Data TypeSourcePurpose
YouTube video URLsPages you visit on youtube.comImport videos into NotebookLM
YouTube video titlesPages you visit on youtube.comName notebooks and detect duplicates
YouTube channel namesPages you visit on youtube.comMatch citation sources in NotebookLM
Import preferencesUser-configured settingsRemember your preferred import options
API key (optional, BYOK)User-provided in settingsAI-powered summaries (if enabled by user)

We do NOT collect: names, email addresses, browsing history, IP addresses, cookies, location data, financial information, or any other personally identifiable information (PII).

2. How We Process Data

All data processing happens entirely within your browser using Chrome's extension APIs. Specifically:

  • Video metadata extraction — When you click the import button on a YouTube page, the extension reads the video URL and title from the current page DOM. No data is scraped in the background.
  • Duplicate detection — Video URLs are compared against a locally stored cache to prevent reimporting the same content.
  • Citation source matching — Channel names and video titles are compared locally to resolve citation links within NotebookLM.
  • AI summaries (optional, BYOK only) — If you provide your own API key (OpenAI or Anthropic), video metadata is sent directly from your browser to the AI provider you selected. VideoLM does not proxy, intercept, or store these API calls. Your API key is stored locally in Chrome's storage and is never transmitted to us.

3. Data Storage

VideoLM stores data exclusively on your device using Chrome's built-in storage APIs:

DataStorage MethodRetention

Duplicate detection cachechrome.storage.localAutomatically trimmed to most recent entries
Source index (citation matching)chrome.storage.localAutomatically trimmed to most recent entries
Import session statechrome.storage.sessionCleared when browser session ends
BYOK API key (if provided)chrome.storage.localUntil you remove it or uninstall

No external databases, cloud storage, or remote servers are used. Uninstalling VideoLM removes all stored data.

4. Data Sharing

VideoLM does not share, sell, trade, or transfer your data to any third party.

The extension communicates only with the following domains, solely to perform its core functionality at your explicit request:

  • youtube.com — To read video metadata from YouTube pages you visit (URLs, titles, channel names)
  • notebooklm.google.com — To import video content into your Google NotebookLM notebooks
  • api.openai.com or api.anthropic.com (optional, BYOK only) — Only if you provide your own API key and initiate an AI summary. These requests go directly from your browser to the provider; VideoLM does not intermediate.

No data is sent to any analytics service, advertising network, data broker, or any server operated by VideoLM.

5. Data Retention and Deletion

  • Session data (active import state) is automatically cleared when you close your browser.
  • Local data (preferences, caches) persists until you uninstall the extension or manually clear extension data via chrome://extensions.
  • BYOK API key can be removed at any time from the extension's Settings page.
  • To delete all data: Uninstall VideoLM from Chrome. All locally stored data is immediately and permanently removed.

7. Security

  • All communication with youtube.com and notebooklm.google.com occurs over HTTPS.
  • BYOK API keys are stored in Chrome's encrypted local storage and are never logged, transmitted to us, or exposed to web pages.
  • The extension does not execute any remote code. All functionality is bundled within the extension package.
  • Source code is publicly available at github.com/WXY395/VideoLM.

8. Children's Privacy

VideoLM is not directed at children under the age of 13. We do not knowingly collect any personal information from children.
