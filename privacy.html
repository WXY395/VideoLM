VideoLM Privacy Policy

Last updated: May 3, 2026

VideoLM does not sell personal data, run advertising tracking, or collect analytics.

VideoLM processes YouTube and NotebookLM data only to provide its import, export, duplicate detection, license, quota, and optional AI features.

1. Data We Process

We do not intentionally collect names, email addresses, payment card data, location data, advertising identifiers, or unrelated browsing history.

2. Local Storage

VideoLM stores the following data on your device using Chrome storage APIs:

  • Import preferences and export settings.
  • Duplicate detection cache and source index.
  • Temporary import session state.
  • Optional BYOK API key if you enter one.
  • License key, install id, entitlement token, plan, and mirrored quota usage.

3. Backend Communication

VideoLM may communicate with the VideoLM backend for these limited purposes:

  • Registering an extension install for entitlement and quota checks.
  • Validating a license key and refreshing Free/Pro plan status.
  • Reserving and refunding import quota for the official extension flow.
  • Providing optional bundled Pro AI processing, if enabled.

The backend is not used for advertising, analytics, data brokerage, or unrelated user profiling.

4. Third-Party Services

The extension communicates with the following services only as needed for user-facing functionality:

  • youtube.com: read YouTube page metadata when you use VideoLM on YouTube.
  • notebooklm.google.com: import sources into your NotebookLM notebooks and read visible NotebookLM content for export features.
  • VideoLM backend: validate license, quota, entitlement, and optional bundled Pro AI features.
  • OpenAI or Anthropic: only if you provide your own API key or use a bundled Pro AI feature that routes through the VideoLM backend.

5. Data Sharing

VideoLM does not sell, rent, or transfer your data to advertisers, data brokers, or analytics providers.

Data is shared only when necessary to provide the extension features described above, comply with law, protect against abuse, or operate the service infrastructure.

6. Retention and Deletion

  • Local settings and caches remain until you remove them, clear Chrome extension data, or uninstall VideoLM.
  • Session state is temporary and cleared by Chrome session storage.
  • License and quota records may remain on the backend while needed to provide entitlement, quota enforcement, fraud prevention, support, and legal compliance.
  • You may remove your BYOK API key from Settings at any time.

8. Security

  • All network communication uses HTTPS.
  • BYOK API keys are stored in Chrome storage and are not sent to VideoLM unless you explicitly use a VideoLM-hosted AI feature that requires it. Current BYOK calls are made directly to the selected provider from your browser.
  • The extension does not execute remote code. Extension functionality is bundled in the package.
  • Source code is available at github.com/WXY395/VideoLM.

9. Children

VideoLM is not directed at children under 13, and we do not knowingly collect personal information from children.

10. Changes

If this policy changes materially, the "Last updated" date will be updated. Continued use of VideoLM after changes means you accept the updated policy.
